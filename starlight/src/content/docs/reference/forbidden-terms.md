The docs should not use old roadmap terms as current product language.

| Avoid this kind of language | Use |
|---|---|
| Intermediate version labels between V1 and V2 | `V2`. |
| Open-ended "future version" labels | `V3` or a concrete later version. |
| Static-runtime architecture labels | `Supabase-backed internal platform` or `data-driven stream`. |
| Old canonical-file artifact language | `frame submission` or `frame metadata`. |
| Date-only stream routes | `/{agent_slug}/{yyyymmdd}/`. |
| User-profile stream namespaces | external tenant routing, V3 only if revived. |

Historical terms belong in git history, not in live docs.
